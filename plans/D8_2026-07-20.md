### 第3步：二级指针实战（25min）[done]

**讲解内容：**

**第1段（从已知类比引入）：**
你已经知道：`int a = 5;` → 变量 a 存值 5
`int *p = &a;` → 指针 p 存 a 的地址
现在问：如果要修改 p 本身（比如让 p 指向另一个变量），怎么办？
答：需要另一个指针来指向 p——这就是二级指针。

```
内存布局图：
a  = [5]         ← a 的值
   地址：0x100
p  = [0x100]     ← p 存的是 a 的地址（&a）
   地址：0x200
pp = [0x200]     ← pp 存的是 p 的地址（&p）
   地址：0x300

关系：
*p  = a  = 5         （一级解引用 → 获得 a）
*pp = p  = &a        （二级解引用 → 获得 p，即 a 的地址）
**pp = *p = a = 5    （三级解引用 → 获得 a 的值）
```

**第2段（实战意义）：**
一级指针 → 函数内修改外部变量的值（swap 传 &a）
二级指针 → 函数内修改外部指针的值（链表头指针修改、内存分配）

**英语科普：** pointer = 指针（point=指向 + er=者）。double pointer = 二级指针（double=双重的，两层指向）。dereference = 解引用（de=去除，reference=引用 → 去掉一层"指"）

**安全彩蛋（≤30s）：** 二级指针在堆漏洞利用中常用于"任意地址写"——攻击者通过修改指针的指针改写关键内存（如 GOT 表项），拿到代码执行权限

**📝 出题（第1层 — 填空 + 画图）：**

```c
    printf("调用后：p = %p\n", (void*)p);
    
    return 0;
}
```

### 第4步：栈帧基础（20min）

**讲解内容：**

**第1段（类比引入）：**
想象你在一叠便签纸上写字：
- main() 写了便签1 → 放在桌上
- main 调用 func1() → 拿一张新便签2，放在便签1上面
- func1 调用 func2() → 再拿一张新便签3，放最上面
- func2 返回 → 撕掉便签3
- func1 返回 → 撕掉便签2

这就是栈帧：每调一个函数，叠一张便签（一个栈帧）。后进先出。

**第2段（详细机制）：**
栈从高地址向低地址生长（画箭头↓）
每个栈帧包含：
1. 函数的参数（或参数 copy）
2. 局部变量
3. 返回地址（函数结束后回到哪）
4. 保存的寄存器（调用者的现场）

```
高地址
┌─────────────────┐
│   main 的栈帧     │  a = 1
├─────────────────┤
│   func1 的栈帧    │  x = 1, y = 2
├─────────────────┤
│   func2 的栈帧    │  z = 3
└─────────────────┘
低地址（栈顶，最新调用）
```

**英语科普：** stack = 栈（一叠/一摞，如 stack of papers）。frame = 帧/框架（一个独立的"格"）。Stack pointer (SP) = 栈指针寄存器，总是指向当前栈顶

**安全彩蛋（≤30s）：** 栈溢出攻击的核心就是往栈帧里写入超出缓冲区大小的数据，覆盖相邻的返回地址——程序执行完当前函数后"回到"攻击者指定的地址执行恶意代码

**📝 出题（第1层 — 画图 + 填空）：**

```c

}

    func2();
}

int main() {
    int a = 1;
    printf("main: a = %d\n", a);
    func1(a);
    return 0;
}
```

1. 画调用栈图：main → func1 → func2（叠三层），标出每个帧里的局部变量
2. 回到主控台最后一行为什么看到 func2 的输出先出现，然后才是 func1 的输出？（从栈帧弹出顺序解释）

### 第5步：指针与数组等价性（15min）

**讲解内容：**

**第1段（已知推导）：**
你学过：`arr[0]` 是数组第一个元素
编译器看到 `arr[0]`，实际翻译成 `*(arr + 0)`——即"从 arr 地址往后数 0 个元素，取出那个值"
所以：`arr[i]` = `*(arr + i)`

**第2段（延伸理解）：**
```c
int arr[5] = {10, 20, 30, 40, 50};
// arr[2]     ===  *(arr + 2)     === 30
// arr        ===  &arr[0]        === 首元素地址
// arr + 2    ===  &arr[2]        === 跳过 2 个 int = 8 字节
```

**第3段（sizeof 陷阱复习）：**
数组名在大多数表达式中"退化"为首元素指针
**例外：** `sizeof(arr)`——只有定义数组的作用域里才知道完整大小
一旦传给函数：`void func(int arr[])` → 这里的 `arr` 其实是 `int *arr`（语法糖）

**关联 D7：** 这就是为什么 `my_strcpy(dest, src, dest_size)` 必须由调用者传 dest_size——函数内部根本不知道 dest 有多大

**英语科普：** equivalence = 等价（equi=相等，val=价值，ence=名词）。syntactic sugar = 语法糖（syntactic=句法的，sugar=糖 → 让代码"变甜"的语法简化）。decay = 退化（de=向下，cay=落下 → "降级"）

**📝 出题（第1层 — 预测输出）：**

```c

    printf("函数内 sizeof(arr) = %lu\n", sizeof(arr));

    printf("sizeof(a) = %lu\n", sizeof(a));

    // 第2题
    printf("a[5] = %d\n", a[5]);
    printf("*(a + 5) = %d\n", *(a + 5));
    
    return 0;
}
```

**问题：**
1. 第1题：两个 sizeof 输出分别是多少？解释为什么不一样
2. 第2题：`a[5]` 和 `*(a + 5)` 输出相同吗？`a[5]` 内部被编译器翻译成了什么？
3. **追问（推导题）：** `5[a]` 合法吗？如果能，它的值是多少？

### 第6步：递归函数（25min）[done]

**讲解内容：**

**第1段（类比引入）：**
递归 = 俄罗斯套娃：打开一个套娃，里面还有一个小一点的套娃，再打开，还有更小的……
直到最后一个不能再打开的（base case），然后从内向外组装回来。

数学上：阶乘 factorial(n) = n × factorial(n-1)，且 0! = 1

**第2段（代码 + 栈帧展开）：**

```c
#include <stdio.h>

int factorial(int n) {
    if (n == 0) {         // base case：终止条件
        return 1;
    }
    return n * factorial(n - 1);  // recursive case：自己调自己
}

int main() {
    int result = factorial(4);
    printf("4! = %d\n", result);
    return 0;
}
```

**画图：factorial(4) 的栈帧变化**

```
展开（递推）阶段 — 每调一次，压一层栈：
factorial(4) 调用 factorial(3)
  → factorial(3) 调用 factorial(2)
    → factorial(2) 调用 factorial(1)
      → factorial(1) 调用 factorial(0)
        → factorial(0) 返回 1    ← base case 命中！
        
回退（回归）阶段 — 每返回一次，弹一层栈：
      ← factorial(0) = 1
    ← factorial(1) = 1 * 1 = 1
  ← factorial(2) = 2 * 1 = 2
← factorial(3) = 3 * 2 = 6
← factorial(4) = 4 * 6 = 24
```

**第3段（必要条件）：**
1. Base case：一定有终止条件（不满足时停止递归）
2. Recursive case：每次递归调用必须向 base case 靠近（n 变小→最终到 0）
3. 缺少任意一个 → 无限递归 → Stack Overflow

**英语科普：** recursion = 递归（re=再次，curs=跑，ion=名词 → 反复跑回来）。base case = 基准情形（base=基础，case=情形）。stack overflow = 栈溢出（stack=栈，overflow=溢出）

**安全彩蛋（≤30s）：** "Stack Overflow"这名字今天理解了吧？——递归无限调用堆满栈，真正的栈溢出攻击和这类似：填充栈帧覆盖返回地址

### 第7步：回调函数与内存分配的关系（10min）[D9 铺垫]

**讲解内容：**

**第1段（从已知概念引入）：**
你 D7 写过 `my_strcpy(dest, src, dest_size)`——调用者传给你 dest_size，你在函数里用。
这是"函数在内部使用调用者提供的数据"。

回调函数反过来：**你在函数里，调用调用者提供的一个函数。**
比如：排序函数不知道怎么比较两个元素，但你传给它一个"比较函数"。

**第2段（回调与内存的关系）：**
假设你有一个排序函数 `sort(arr, n, compare_fn)`：
- `compare_fn` 是一个回调函数——每次比较两个元素时，sort 调用它
- 问题：回调函数需要访问一些数据（比如按名字排序还是按年龄排序），这些数据谁分配？谁释放？
  → 这需要一种"内存契约"：回调的提供者负责管理回调所需的资源

**英语科普：** callback = 回调（call=调用，back=回 → 调用回来）。register callback = 注册回调（register=登记 → 告诉系统"如果有事件，叫我"）

**⏰ 标注：** "回调函数的声明语法（函数指针声明、typedef 化简）是 D9（明天）的重点，今天只要理解概念——'不是我调它，是它回调我'"

**📝 出题（第1层 — 填空题）：**
