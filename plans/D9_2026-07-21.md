### 第1步：函数指针声明语法（20min）

**讲解内容：**

**第1段：从已知知识出发**

我们已经知道：数组名在表达式中会退化为首元素指针（D7学过的数组退化）。

**函数名也会退化**——函数名在表达式中代表函数的入口地址。

那怎么声明一个"能存函数地址的变量"？

**第2段：声明模仿使用原则**

C 语言里有一个核心原则：**声明模仿使用**。

意思是：你在使用（写代码）时怎么用这个变量，声明时就怎么描述它。

比如：
- `int *p;` → `*p` 是一个 int → p 是一个 int 指针
- `int arr[5];` → `arr[5]` 是一个 int → arr 是一个 int 数组

现在我要声明一个指针 p，它能指向一个函数。

假设有一个函数：`int add(int a, int b)`。
如果我通过指针 p 来调它，代码怎么写？

```c
int result = (*p)(3, 5);
```

或者更简洁（函数指针的特殊语法）：

```c
int result = p(3, 5);
```

**声明模仿使用：** 把使用表达式里的 `p` 换成 `(*p)`（因为声明时要强调"这是一个指针"）：

```c
int (*p)(int, int);
```

来逐段拆解：
```
int  (*p)    (int, int);
 │     │        │
 │     │        └─ 参数类型列表
 │     └─ 变量名（带 * 表示是指针）
 └─ 返回类型
```

**第3段：为什么必须加括号？**

对比：
```c
int *p(int, int);   // 这是一个函数，返回 int*
int (*p)(int, int); // 这是一个指针，指向返回 int 的函数
```

函数调用运算符 `()` 优先级高于 `*`——不加括号的话，先结合 `p(int, int)` 变成函数声明，`*` 修饰的是返回值。

**第4段：赋值和调用**

```c
int add(int a, int b) {
    return a + b;
}

int main() {
    int (*p)(int, int);  // 声明：p 是一个函数指针
    p = add;             // 赋值：函数名退化，等价于 &add
    
    // 三种调用方式都是等价的
    printf("%d\n", p(3, 5));     // 直接调（推荐）
    printf("%d\n", (*p)(3, 5));  // 显式解引用
    printf("%d\n", add(3, 5));   // 直接调函数
}
```

函数名 `add` 在表达式中**自动退化**为函数指针，类似数组名退化为数组指针。`p = add` 和 `p = &add` 等价。

**英语科普：**
- **pointer to function** = 指向函数的指针（简称"函数指针"）
- 函数名在英语里叫 **function identifier**
- embedded 里函数指针常用于：**callback**（回调）、**interrupt vector table**（中断向量表）、**state machine dispatch table**（状态机跳转表）

**安全彩蛋（≤30s）：**
> 中断向量表本质上就是一个函数指针数组！把函数指针改成恶意地址→这就是 **vtable hijacking**，嵌入式安全里经典的攻击手法。后续 W4 中断专题时会展开。

**📝 第1题：填空——函数指针声明（第1层·填空）**

```c

    return x * y;
```

### 第2步：typedef 化简函数指针类型（10min）

**讲解内容：**

**第1段：D4 的痛，今天不重犯**

D4 讲 typedef 时，我犯过错误——先给了简化模型 `typedef X Y`，结果到函数指针时崩塌了。

今天直接上**完整范式**。

**第2段：完整范式**

```c
typedef int (*OpFunc)(int, int);
```

拆解步骤（对照普通变量声明）：

```c
// 第一步：声明一个函数指针变量
int (*p)(int, int);

// 第二步：把变量名 p 替换成类型别名
// p → OpFunc
typedef int (*OpFunc)(int, int);

// 第三步：用类型别名声明
OpFunc p;  // 等价于 int (*p)(int, int);
```

**规则一句话：typedef 去掉变量名，剩下的就是类型。**

```c
// 普通类型
int arr[5];              // arr 是 int[5]
typedef int Arr5[5];     // Arr5 是类型 int[5]
Arr5 a;                  // a 是 int[5]

// 函数指针
int (*fp)(int, int);         // fp 是函数指针
typedef int (*FP)(int, int); // FP 是函数指针类型
FP fp;                       // fp 是函数指针
```

**英语科普：**
- **typedef** = type + definition（类型定义别名）
- 在嵌入式里 typedef 函数指针是**非常常见的**——很多 HAL 库的初始化结构体里都包含函数指针成员

### 第3步：回调函数实战——计算器（20min）

**讲解内容：**

**第1段：从概念到代码**

D8 我们说了"回调 = 把函数传给别人调"。今天把它写出来。

场景：写一个计算器函数，它接受两个整数 + 一个运算操作符，自动执行对应运算。

```c
// 定义运算函数
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }
int divide(int a, int b) { return a / b; }

// 回调模式：接受函数指针作为参数
int calc(int a, int b, int (*op)(int, int)) {
    return op(a, b);  // 在内部调用传入的函数
}

int main() {
    printf("%d\n", calc(10, 5, add));   // 15
    printf("%d\n", calc(10, 5, sub));   // 5
    printf("%d\n", calc(10, 5, mul));   // 50
    printf("%d\n", calc(10, 5, divide)); // 2
}
```

这样 `calc` 函数**不需要知道具体运算逻辑**——你传什么函数进来，它就做什么运算。这就是回调的核心思想。

**第2段：回调的优点**

1. **解耦：** calc 只负责"调用传进来的函数"，不负责具体实现
2. **可扩展：** 要加取模运算，不需要改 calc，只需要写一个新函数 `mod`
3. **复用：** 同样的模式可以用在排序、查找、事件处理等场景

**第3段：嵌入式场景联想**

在嵌入式里，回调无处不在：
- 定时器中断 → 到时间了调用你注册的回调
- GPIO 外部中断 → 引脚电平变化了调用你的回调
- HAL 库的初始化 → 结构体里填回调函数

**英语科普：**
- **callback** = call + back（回调→先注册，回头再调）
- 类似"call me back"（回电话）——先留下号码，合适时对方打回来
- **dispatch table** = 分发表，嵌入式状态机的核心实现方式（中断向量表就是一个 dispatch table）

**安全彩蛋（≤30s）：**
> 回调函数如果被篡改（比如缓冲区溢出覆盖了函数指针），程序就会跳转到攻击者指定的地址执行——这叫 **function pointer overwrite**，是嵌入式系统安全中非常严重的漏洞类型。所以嵌入式中函数指针变量常加 `const` 保护。

### 第4步：qsort——标准库的回调应用（20min）

**讲解内容：**

**第1段：qsort 是什么**

C 标准库提供了一个通用的排序函数——`qsort`（quick sort 的缩写）。它**不关心**你要排序的是什么类型的数据，它只负责"按你给的规则"排。

为什么能做到通用？因为**回调**。

你传入一个比较函数，告诉 qsort "什么样的顺序算递增"，qsort 内部在比较时调你的回调。

**第2段：qsort 的函数签名**

```c
void qsort(
    void *base,      // 数组起始地址
    size_t num,      // 元素个数
    size_t size,     // 每个元素的大小（字节数）
    int (*compar)(const void *, const void *)  // 比较函数（回调！）
);
```

比较函数必须返回：
- **负数** → a 排在 b 前面（a < b）
- **零** → a 等于 b
- **正数** → a 排在 b 后面（a > b）

**第3段：写一个比较函数**

```c
int cmp_int(const void *a, const void *b) {
    // 从 void* 转回 int*
    int ia = *(const int*)a;
    int ib = *(const int*)b;
    
    // 返回差值
    if (ia < ib) return -1;
    if (ia > ib) return 1;
    return 0;
}

// 或者更简洁的版本（注意边界！int 差值可能溢出）
int cmp_int_simple(const void *a, const void *b) {
    return *(const int*)a - *(const int*)b;
}
```

**为什么用 void*？** 因为 qsort 不知道你要排什么类型——int、double、结构体都可能。void* 是通用指针，"你给我数据地址和比较逻辑，排序逻辑我来做"。

**英语科普：**
- **qsort** = quick sort（快速排序）
- **comparator** = 比较器（comparative 的比较级形式）
- **void*** 在英语里叫 **generic pointer**（泛型指针），`generic` = 通用的、泛化的

**第4段：完整代码示例**

```c
#include <stdio.h>
#include <stdlib.h>

int cmp_int(const void *a, const void *b) {
    int ia = *(const int*)a;
    int ib = *(const int*)b;
    return (ia > ib) - (ia < ib);  // 安全减法（不会溢出）
}

int main() {
    int arr[] = {42, 7, 15, 3, 88, 1, 26};
    int n = sizeof(arr) / sizeof(arr[0]);
    
    qsort(arr, n, sizeof(int), cmp_int);
    
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
    printf("\n");
    // 输出：1 3 7 15 26 42 88
    return 0;
}
```

**安全彩蛋（≤30s）：**
> 嵌入式面试题：如果比较函数写出 bug（如 `return *a - *b` 但 a<b 时负数比正数范围大），排序结果会乱——这在嵌入式里可能导致**状态机跑到非法状态**。写回调函数时边界测试必须严格！

### 第5步：二分查找（25min）

**讲解内容：**

**第1段：为什么讲它？**

排序和查找是一对好兄弟。排序完通常要查找——二分查找是最高效的查找方法（`O(log n)`）。

而且 C 标准库的 `bsearch` 也是用回调的，学完 qsort 后 bsearch 几乎不用学就会用。

**第2段：二分查找的原理**

一个类比：**猜数字游戏**

我想一个 1-100 之间的数，你猜，我只能告诉你"大了"或"小了"。

最优策略是什么？每次都猜中间！

```
范围 1-100 → 猜 50，说"大了"
范围 1-49  → 猜 25，说"小了"
范围 26-49 → 猜 37，说"大了"
范围 26-36 → 猜 31，...
```

每次砍掉一半范围 → log₂(100) ≈ 7 次就找到。

**核心思想：**
```
while (范围不空) {
    mid = 范围的中间位置
    if (arr[mid] == 目标) → 找到了！
    if (arr[mid] < 目标) → 去右半边找
    if (arr[mid] > 目标) → 去左半边找
}
```

**第3段：完整实现**

```c
int binary_search(int arr[], int n, int target) {
    int left = 0;
    int right = n - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防止 left+right 溢出
        
        if (arr[mid] == target)
            return mid;       // 找到了，返回下标
        else if (arr[mid] < target)
            left = mid + 1;   // 去右边找
        else
            right = mid - 1;  // 去左边找
    }
    
    return -1;  // 没找到
}
```

**注意：** `mid = (left + right) / 2` 有溢出风险（当数组足够大时），推荐 `left + (right - left) / 2`。

### 第6步：可变参数 va_list（20min）

**讲解内容：**

**第1段：问题引入**

```c
printf("%d %f %s", 42, 3.14, "hello");
```

printf 能接受**任意数量**的参数——这在 C 里怎么实现的？我们平时的函数参数数量是固定的。

答案是 **va_list** —— C 标准库提供的可变参数机制。

**第2段：四个宏**

```c
#include <stdarg.h>

va_list ap;          // 声明一个"参数列表"变量
va_start(ap, n);     // 初始化：从第 n 个参数之后开始取
va_arg(ap, type);    // 取下一个参数（type 指明类型）
va_end(ap);          // 清理
```

**第3段：完整示例——变参求和函数**

```c
#include <stdio.h>
#include <stdarg.h>

int sum(int count, ...) {  // ... 代表可变参数
    va_list ap;
    int total = 0;
    
    va_start(ap, count);   // 从 count 后面开始取
    
    for (int i = 0; i < count; i++) {
        total += va_arg(ap, int);  // 依次取 int 参数
    }
    
    va_end(ap);
    return total;
}

int main() {
    printf("%d\n", sum(3, 10, 20, 30));   // 60
    printf("%d\n", sum(5, 1, 2, 3, 4, 5)); // 15
}
```

**关键约束：**
1. 至少有一个**命名参数**（`count`）——否则 va_start 无法知道从哪里开始
2. 你必须知道每个参数的类型——va_arg 不会自动判断类型（这就是为什么 printf 用格式字符串）
3. 你必须知道参数个数——`...` 不会告诉你还有几个参数没取

**第4段：printf 的底层思路**

printf 的实现思路（简化版）：

```c
void my_printf(const char *fmt, ...) {
    va_list ap;
    va_start(ap, fmt);
    
    for (int i = 0; fmt[i] != '\0'; i++) {
        if (fmt[i] == '%') {
            i++;  // 跳过 %
            switch (fmt[i]) {
                case 'd': {
                    int val = va_arg(ap, int);
                    // 把 val 转成字符串输出
                    break;
                }
                case 'f': {
                    double val = va_arg(ap, double);
                    // 把 val 转成字符串输出
                    break;
                }
                // ...
            }
        } else {
            putchar(fmt[i]);  // 普通字符直接输出
        }
    }
    
    va_end(ap);
}
```

**安全彩蛋（≤30s）：**
> printf 的可变参数是经典漏洞来源——格式字符串里写 `%s` 但不传对应参数 → 它会从栈上随便读个地址当字符串输出，这叫 **format string vulnerability**。嵌入式中被用来读取栈上的敏感信息（密码、密钥）。

> ⏰ 这是 W3 文件 IO / W5 安全专题的内容，现在只需要知道：printf 内部用 va_list 读取参数，类型靠格式字符串中的 `%d` `%f` 等来匹配。如果你传了 `%d` 但没给足参数，它会从栈上读垃圾值。

**📝 第6题：填空+自写——简易 my_printf（第2层·自写关键行）**

```c
#include <stdio.h>
#include <stdarg.h>

void my_printf(const char *fmt, ...) {
    va_list ap;
    ______(ap, fmt);        // 填空1：初始化
    
    for (int i = 0; fmt[i] != '\0'; i++) {
        if (fmt[i] == '%') {
            i++;  // 跳过 %
            switch (fmt[i]) {
                case 'd': {
                    int val = ______(ap, int);  // 填空2：取一个 int
                    printf("%d", val);          // 简化：直接调标准 printf
                    break;
                }
                case 'c': {
                    char ch = ______(ap, int);  // 填空3：char 在可变参数中会被提升为 int
                    putchar(ch);
                    break;
                }
                default:
                    putchar(fmt[i]);
            }
        } else {
            putchar(fmt[i]);
        }
    }
    
    ______(ap);  // 填空4：清理
}

int main() {
    my_printf("姓名: %c, 年龄: %d, 分数: %d\n", '文', 21, 95);
    // 预期输出：姓名: 文, 年龄: 21, 分数: 95
    return 0;
}
```

> **代码审查红线检查：** va_arg 类型必须匹配实际传入的类型，否则读取会错（如传 float 但用 va_arg(ap, double) 是对的，因为 float 自动提升为 double）
