**问题：**
(1) 上面代码的输出是什么？（两个数分别多少）

### ③ 函数指针基础（20min）

**讲解内容：**

**第一步：从数组名退化为指针引入函数名也是地址**

先回顾学生已知的：`arr[i]` 等价于 `*(arr + i)`，数组名在函数传参时退化为指针。

我问一个问题让学生自己推导："函数名是不是也是地址？我们之前怎么调用函数的？"

```c
int add(int a, int b) { return a + b; }
// 直接调用
int r = add(3, 5);
// 那我问：add 这个标识符本质上是什么？
// 答：是代码段里那个函数的第一条指令的起始地址
```

**第二步：函数指针声明（完整范式，不先简化）**

语法拆解：

```
int    (*pf)       (int, int);
└┬┘   └┬┘         └──┬───┘
返回  是指针       参数列表
类型          名字叫 pf
```

逐段讲解：
1. `pf` 是一个变量名（先找到名字）
2. `*pf` 表示 pf 是指针（括起来因为 `*` 优先级低）
3. `(*pf)(int, int)` 表示这个指针指向一个接受两个 int 参数、返回 int 的函数
4. `int (*pf)(int, int)` 完整声明：pf 是一个指向"接受两个 int、返回 int 的函数"的指针

**第三步：三种等价调用方式**

```c
int add(int a, int b) { return a + b; }
int (*pf)(int, int);

pf = add;          // 方式一：直接赋值（函数名退化为地址）

// 调用——三种写法完全等价
int r1 = pf(3, 5);       // 写法1：直接用指针名调用
int r2 = (*pf)(3, 5);    // 写法2：解引用后再调用
int r3 = add(3, 5);      // 写法3：直接用函数名
```

**英语科普：** pointer to function = 指向函数的指针。callback = 回调（call=叫回来，back=回 → "叫回来"：你先定一个动作，等时机到了系统叫你回来执行）

**安全彩蛋（≤30秒）：** 中断向量表本质上就是一个函数指针数组。如果攻击者能改写这个数组（比如通过缓冲区溢出覆盖到向量表区），就能让 CPU 在中断发生时跳转到恶意代码。这叫"函数指针劫持"。

**第四步：typedef 化简（完整拆解）**

```c
// 原始版本——声明一个函数指针变量
int (*pf)(int, int);

// 用 typedef 给它起个别名
// 写法：把变量名换成类型名，前面加 typedef
typedef int (*OpFunc)(int, int);

// 现在可以用 OpFunc 来声明变量了
OpFunc pf1;        // 等价于 int (*pf1)(int, int);
OpFunc pf2;        // 等价于 int (*pf2)(int, int);
```

**⭐ 拆解技巧：** typedef 函数指针类型时，"找到变量名 → 换成类型名 → 前面加 typedef"

```
// 原变量声明：
int (*pf)(int, int);
//    ↑ 找到变量名 pf
//    ↓ 换成类型名 CalcFunc
int (*CalcFunc)(int, int);
//  ↑ 前面加 typedef
typedef int (*CalcFunc)(int, int);
```

### ④ qsort + 回调函数（25min）

**讲解内容：**

**qsort 的函数签名：**

```c
#include <stdlib.h>

void qsort(
    void *base,        // 要排序的数组首地址
    size_t nmemb,      // 元素个数
    size_t size,       // 每个元素的字节大小
    int (*compar)(const void *, const void *)  // 比较函数（回调！）
);
```

**逐参数拆解：**
- `base`：数组首地址（`void *` 因为它不知道你存了什么类型的数据）
- `nmemb`：数组里有多少个元素
- `size`：每个元素占多少字节（用 `sizeof(元素类型)` 传）
- `compar`：一个函数指针！指向你的比较函数。qsort 在排序过程中会反复调用这个函数来比较两个元素

**回调机制解释：**
- qsort 自己不知道你的数组里存的是什么类型（所以 base 是 void*）
- qsort 靠你传给它的 compar 来决定两个元素谁大谁小
- qsort 在需要比较时：`compar(p1, p2)` → 你的函数返回负数/0/正数 → qsort 判断大小

**比较函数的写法模板：**

```c
// 比较两个 int（升序）
int cmp_int_asc(const void *a, const void *b) {
    int va = *(const int *)a;  // 先把 void* 强转为 int* 再解引用
    int vb = *(const int *)b;
    if (va < vb) return -1;    // a < b → 负数
    if (va > vb) return 1;     // a > b → 正数
    return 0;                   // a == b → 0
}

// 比较两个 int（降序）——改一下返回值方向即可
int cmp_int_desc(const void *a, const void *b) {
    int va = *(const int *)a;
    int vb = *(const int *)b;
    if (va > vb) return -1;    // 反过来了
    if (va < vb) return 1;
    return 0;
}
```

**简化写法（嵌入式里常见但严谨性稍差）：**

```c
int cmp_int_asc(const void *a, const void *b) {
    return *(int *)a - *(int *)b;   // 升序
    // return *(int *)b - *(int *)a;  // 降序
}
```
⚠️ 缺点：减法可能整数溢出（但大多数笔试题里这么写）

**调用 qsort：**

```c
int arr[] = {5, 3, 8, 1, 9, 2, 7, 4, 6};
int n = sizeof(arr) / sizeof(arr[0]);

qsort(arr, n, sizeof(int), cmp_int_desc);

// 打印排序结果
for (int i = 0; i < n; i++) {
    printf("%d ", arr[i]);
}
// 输出：9 8 7 6 5 4 3 2 1
```

**关联 D7 综合项目1：** 在项目1里我们用冒泡排序手写了按平均分降序。如果用 qsort 只需要写一个比较函数，一行调用就搞定。

**英语科普：** qsort = quick sort（快速排序）。compare = 比较（com-=共同，par=匹配/配对，-e → "放在一起配对比"）

### ⑤ 二分查找（20min）

**讲解内容：**

**原理：**
- 前提条件：数组必须已排序（接上面 qsort 的结果直接用）
- 思想：low 左指针、high 右指针、mid 中间指针
  - 如果 arr[mid] == target → 找到了
  - 如果 arr[mid] < target → 目标在右半区，low = mid + 1
  - 如果 arr[mid] > target → 目标在左半区，high = mid - 1

**代码模板：**

```c
int binary_search(int arr[], int n, int target) {
    int low = 0;
    int high = n - 1;

    while (low <= high) {
        int mid = low + (high - low) / 2;  // 防溢出写法

        if (arr[mid] == target) {
            return mid;       // 找到了，返回下标
        } else if (arr[mid] < target) {
            low = mid + 1;    // 目标在右半区
        } else {
            high = mid - 1;   // 目标在左半区
        }
    }

    return -1;  // 没找到
}
```

**重点讲解 mid 写法：**

```c
// 两种写法：
int mid = (low + high) / 2;           // 写法1：如果 low+high 大于 INT_MAX 会溢出
int mid = low + (high - low) / 2;     // 写法2：不会溢出，因为 high - low 最大为 n
```

**英语科普：** binary = 二进制的（bi- = 二），search = 搜索。所以 binary search = 二分搜索。logarithmic time = 对数时间（O(log n) — 每次砍掉一半数据）

### ⑥ 可变参数 va_list（15min）

**讲解内容：**

**问题引入：**
- `printf("Hello")` — 1个参数
- `printf("%d + %d = %d", a, b, a + b)` — 4个参数
- **printf 是怎么做到传不同数量参数的？**

**原理：**
- 函数参数在调用时从右往左压栈
- 所有参数连续排列在栈上
- va_list 通过指针偏移逐个取出参数
- 调用者必须告诉你有多少个参数（printf 用 %d/%s 的数量来标识）

**四步曲模板：**

```c
#include <stdarg.h>   // 必须包含这个头文件

int sum(int n, ...) {          // ... 表示"后面有可变数量的参数"
    va_list args;               // ① 声明一个参数列表变量
    va_start(args, n);          // ② 初始化：从 n 后面的第一个参数开始

    int total = 0;
    for (int i = 0; i < n; i++) {
        total += va_arg(args, int);  // ③ 取出下一个 int 类型的参数
    }

    va_end(args);               // ④ 清理

    return total;
}

// 调用
int result = sum(4, 10, 20, 30, 40);  // result = 100
```

**逐步讲解每行：**
1. `va_list args` — 声明一个参数列表的迭代器（你可以理解为一个"当前读取位置的指针"）
2. `va_start(args, n)` — 初始化 args，让 args 指向 n 后面第一个参数的位置
3. `va_arg(args, int)` — 从 args 当前位置取出一个 int，然后 args 前移一个 int 的位置
4. `va_end(args)` — 清理（在有些架构上需要释放资源）

**英语科普：** variable = 可变的（vari-=变化，-able=能）。va_list = variable argument list（可变参数列表）。va_start = variable argument start，va_arg = variable argument（取参数），va_end = variable argument end
