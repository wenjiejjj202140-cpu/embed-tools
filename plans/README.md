# 📋 每日教案生成 SOP

> 目标：每次会话启动时，仅通过读源文件即可产出和之前同等质量的完全体教案
> 不依赖会话间的记忆传递，只靠本文件驱动

---

## 📂 前置读文件（按顺序）

| 序号 | 文件 | 读什么 |
|------|------|--------|
| 1 | LEARNING_PLAN.md | 今天该讲什么主题？作业是什么？ |
| 2 | knowledge.toml | 有哪些 status=review 的知识点？（优先复习） |
| 3 | MEMORY.md | 最近一条「教学反思」— 上次卡在哪？ |
| 4 | MASTER.md | 第四部分「教学流程（4步法）」+ 第三部分「行为准则」 |
| 5 | MEMORY.md 会话启动流程 | 检查 last_knowledge_sync 是否落后 |
| 6 | daily_state.toml | 检查今天 teaching_done 是否 true |

---

## 📄 教案输出结构

必须按以下 5 个区块组织，每个区块都有固定格式。

---

### 区块 A：本课目标（4-6条）

**每条格式：**
```
**目标N：** {一句话概括}
- {要掌握的具体能力1}
- {要掌握的具体能力2}
```

**来源：** LEARNING_PLAN.md 当天计划
**规则：** 每条目标必须是可检验的（学生能说"我掌握了"或"我写出来了"）。不写模糊的"理解xxx"，而是写"能xxx"

---

### 区块 B：教学及出题要求（贯穿全程）

**每条格式：**
```
**要求N：{一句话标题}**
- 来源：{文件名}（{具体章节}）→ "{原文引用}"
- 执行方式：{在当前课中怎么落实}
```

**必须列出的要求（从以下文件提取）：**

| # | 要求 | 来源 |
|---|------|------|
| 1 | 逐点确认 | MEMORY.md（D3反思） |
| 2 | 推导法优先 | MEMORY.md（D5+D6反思） |
| 3 | 复杂语法用完整范式 | MEMORY.md（D4反思） |
| 4 | 支架教学 | MEMORY.md（7/19反思） |
| 5 | 英语科普脱口而出 | MASTER.md §4 步骤1.2 |
| 6 | 安全彩蛋（≤30s） | MASTER.md §4 步骤1.3 |
| 7 | 优先复习 review 标记 | knowledge.toml 出题规则 |
| 8 | 每题答完更新 knowledge.toml | knowledge.toml 出题规则 |
| 9 | 超前知识点标注 "这是第X天的内容" | MASTER.md §3.1 |
| 10 | 代码审查红线（malloc/volatile等） | MASTER.md §4 步骤2.3 |

**规则：** 每条必须带来源文件的**原文引用**，不能自己概括。执行方式要具体到"讲什么内容时怎么做"。

---

### 区块 C：知识点（按顺序）

**每个知识点独立分段，格式：**

```
### 第N步：{知识点名称}（{预估时间}min）

**讲解内容：**

第1段：{从已知知识类比引入}
第2段：{新概念讲解，画图或代码示例}
第3段：{如果语法复杂，拆解到最小粒度}
第4段：{关联已学过的知识点}

**英语科普：** {词汇} = {翻译}（{词根拆解} → {为什么这样命名} → {在嵌入式中的作用}）
> 来源：MASTER.md §4 步骤1.2

**安全彩蛋（≤30s）：** {知识点关联安全方向的一句话科普}
> 来源：MASTER.md §4 步骤1.3

**📝 学完立刻出题：**

{完整代码 + 填空标记 + 问题的完整描述}
标注支架层级：第1层（填空）或第2层（自写关键行）或第3层（从零写）

> 来源：MEMORY.md 支架教学 → 第X层
```

**规则：**
- 每道题必须给出**完整的代码**（含 `#include` 和 `main` 等），不准只写函数体
- 填空用 `______` 标记，不是注释里写"补全你的代码"
- 每题标注所属的**支架层级**
- 英语科普必须提供词根拆解
- 安全彩蛋一句话，不超过30秒内容

---

### 区块 D：预出题汇总

**格式：**

```
## ❓ 预出题汇总

### 复习题（knowledge.toml review 标记优先）
| 知识点 | review 状态 | 题目概要 |
|--------|------------|---------|
| {知识点} | status=review, error_count={N} | {题目一句话} |

### 新课练习题汇总
| 知识点 | 题号 | 支架层级 |
|--------|------|---------|
| {知识点} | 第N题：{题目名} | 第X层 |
```

## 🧭 教学锚定机制（防止偏移）

### 规则：进度标记 + 中断后重读

**执行方式：**

1. **开始教学前：** 读一次完整的教案文件
2. **每讲完一个知识点：** 在教案文件对应的知识点后面写 `[done]`
   ```
   ### 第1步：函数指针基础（20min）[done]
   ### 第2步：qsort + 回调（25min）  ← 当前步
   ```
3. **每次被学生提问打断后**（无论问题是否相关）：
   - 回答完毕 → **立即重读教案文件**
   - 看当前进度标记 → 确认回到主线
4. **如果学生问的是后续知识点：**
   - 回答，然后在教案对应知识点行加标注 `[提前讲了]`
   - 跳到该知识点后，跳过重复讲解

**原理：** 每次重读教案文件 = 导航仪重新计算路线。不靠脑记，靠文件拽回来。

**💡 上课口诀：回答之前先想 3 秒——『这个问题是已学范围还是未来知识点？如果是未来知识点，先标注阶段再回答。』**

## ⚠️ 教案质量检查清单（生成后自查）

生成完毕后逐条检查，如果有一条不是 true，必须重做到 true 才可开始上课：

- [ ] 每个知识点都有对应的题目（逐点确认）
- [ ] 每道题都有完整的代码（含 include 和 main）
- [ ] 复习题从 knowledge.toml 的 review 标记抓取，且标注了 error_count
- [ ] 教学要求区块包含至少 8 条，每条有原文引用和执行方式
- [ ] 题目按支架层级递进（填空→自写→从零写）
- [ ] 英语科普有词根拆解
- [ ] 安全彩蛋在合适知识点插入
- [ ] 课后流程清单完整（5步：含生成笔记）
