# D8 指针进阶与内存模型 + 递归 — 2026/7/20

## 核心知识点

### 1. 位操作宏定义（复习巩固）
```c
#define SET_BIT(reg, n)   ((reg) |= (1 << (n)))   // 置位
#define CLEAR_BIT(reg, n) ((reg) &= ~(1 << (n)))  // 清位
```
- `|=`：只动特定位，其他位保留（有1为1）
- `&=` + `~`：动特定位为0，其他不变
- 括号原因：宏是文本替换，不加括号会因运算符优先级出问题
- `#define 名字(参数) 替换体` —— 宏定义的完整格式

### 2. 数组退化为指针（sizeof陷阱·复习）
```c
int a[100];
printf("%lu", sizeof(a));    // 400 (100 * 4)
void func(int arr[]) {
    printf("%lu", sizeof(arr));  // 8（指针，64位系统）
}
```
- `arr[i]` 内部翻译为 `*(arr + i)`
- 函数参数里的 `[]` 是语法糖，本质就是指针
- `5[a]` 合法 → `*(5 + a)` → 加法可交换

### 3. 二级指针（int **p）
- **声明模仿使用原则：** `int **pp` 表示 "`**pp` 的类型是 `int`"
- 三层盒子图：
  ```
  x = [42]          地址 0x100
  p = [0x100]       地址 0x200   (int*)
  pp = [0x200]      地址 0x300   (int**)
  ```
- 实战意义：函数内修改外部指针的值（**pp → 改 p）
- **声明里的 `*` 是类型声明，表达式的 `*` 是解引用——不是一个东西**

### 4. 栈帧（Stack Frame）
- 每调一个函数压一层栈帧，返回弹一层
- 栈帧里装：局部变量、参数、返回地址
- 栈从高地址向低地址生长（历史约定，堆栈相向最大化内存利用）
- SP（栈指针）直接跳，不是逐层走——编译时算好大小写死在指令里
- 函数返回后栈帧残留但被"回收"——随时可能被下次调用覆盖（悬挂指针）

### 5. 指针与数组等价性
- `arr[i]` ≡ `*(arr + i)`（编译器的翻译规则）
- 指针+偏移的步长由指针类型决定：
  - `int* + 1` → 跳 `sizeof(int)` = 4 字节
  - `char* + 1` → 跳 `sizeof(char)` = 1 字节
  - `double* + 1` → 跳 `sizeof(double)` = 8 字节

### 6. 递归函数
- 递归 = 函数调自己，每层有独立栈帧
- 必要条件：
  - base case（终止条件，如 `n == 0`）
  - 递归公式向 base case 靠近（如 `n-1`）
- 缺失任一条件 → Stack Overflow
- 展开（递推）：逐层压栈
- 回退（回归）：逐层返回

```c
// factorial(n) = n * factorial(n-1), 0! = 1
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}

// sum(n) = n + sum(n-1), sum(0) = 0
int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);
}
```

### 7. 回调函数概念（D9 铺垫）
- 不是我调它，是它回调我
- 类比：留电话给骑手→外卖到了骑手打给你
- 函数指针声明语法是 D9 内容

## 嵌入式注意事项
- ⚠️ MCU 栈空间可能只有 1KB→递归很容易溢出
- ⚠️ 二级指针在堆漏洞利用中用于任意地址写（修改 GOT 表项）
- 💡 排序/遍历场景常用回调函数（qsort 的比较器）

## 英语科普
- **pointer** = 指针（point=指向 + er=者）
- **double pointer** = 二级指针（double=双重的）
- **dereference** = 解引用（de=去除 + reference=引用）
- **stack** = 栈（一摞盘子），**frame** = 帧/格
- **recursion** = 递归（re=再次 + curs=跑 + ion=名词）
- **callback** = 回调（call=调 + back=回）
