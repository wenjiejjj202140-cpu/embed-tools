## 1. 函数指针声明语法

### 声明模仿使用原则

```c
int (*p)(int, int);   // p 是指针，指向"接收两个int、返回int"的函数
int *p(int, int);     // p 是函数，返回 int*
```

**关键规则：** `()` 优先级高于 `*`

- `int *p(int)` → p先跟 `(int)` 结合 → p是函数
- `int (*p)(int)` → `(*p)` 被括号捆在一起先结合 → p是指针

### 赋值和调用

```c
int add(int a, int b) { return a + b; }

int (*p)(int, int);
p = add;              // 函数名退化，等价于 &add

p(3, 5);              // 两种写法等价
(*p)(3, 5);           // 推荐用简洁写法
```

### 函数的本质

函数在代码段存着一串机器指令，函数名 = 这段指令的起始地址标签。函数不是数据，不能复制传参。

## 2. typedef 化简函数指针

```c
typedef int (*OpFunc)(int, int);   // OpFunc 是函数指针类型
OpFunc p;                           // 等价于 int (*p)(int, int)
```

**核心规律：** typedef去掉变量名，剩下的就是类型。

```c
// 普通指针对比
int *p;
typedef int* IntPtr;    // IntPtr = int*
IntPtr p;               // p = int*
```

**注意：** 类型名本身已经包含 `*`，声明变量时不要再加符号。

```c
typedef int (*FP)(int, int);
FP cmp;          // ✅ 正确
FP (*cmp)(int,int);  // ❌ 多了一层指针！
```

## 3. 回调函数

回调 = 把函数地址传给另一个函数，由另一个函数在合适的时机调用。

```c
// 回调模式
int compare(int a, int b, CmpFunc cmp) {
    return cmp(a, b);   // 通过回调调用
}

// 使用时
compare(7, 3, max);   // 传 max 函数作为规则
compare(7, 3, min);   // 传 min 函数作为规则
```

**C 语言传参本质：** 函数不能作为值传递，传参声明 `int cmp(int,int)` 会被编译器自动转为 `int (*cmp)(int,int)`。

## 4. qsort 标准库排序

```c
void qsort(void *base, size_t num, size_t size,
           int (*compar)(const void *, const void *));
```

**比较函数写法：**
```c
int cmp_float(const void *a, const void *b) {
    float fa = *(const float *)a;
    float fb = *(const float *)b;
    if (fa < fb) return -1;   // a 排在 b 前面
    if (fa > fb) return 1;    // a 排在 b 后面
    return 0;
}
```

降序：把 -1 和 1 的位置互换。

## 5. 二分查找

**前提：** 数组必须已排序。

**核心逻辑：**
```c
int binary_search(int arr[], int n, int target) {
    int left = 0, right = n - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防溢出
        
        if (arr[mid] == target)
            return mid;           // 找到了
        else if (arr[mid] < target)
            left = mid + 1;       // 去右边（跳过已查的 mid）
        else
            right = mid - 1;      // 去左边（跳过已查的 mid）
    }
    return -1;  // 没找到
}
```

**死循环陷阱：** left/right 更新必须是 `mid ± 1`，不能只写 `mid`——否则查过的元素又被纳入搜索范围，无限循环。
