# D7 字符串处理 — 2026/7/17

## 核心知识点

### 1. 字符串本质
- C 语言中字符串 = 字符数组 + `\0` 结尾
- `"Hello"` 是字面量，编译时存在 `.rodata`（只读数据段）
- `char s1[] = "Hello"` → 在栈上创建可写副本
- `char *s2 = "Hello"` → 指针指向只读区，修改会 segfault

### 2. 指针步长
- 指针 + N 跳多远取决于指向的类型
- `char*` + 1 → 跳 1 字节
- `int*` + 1 → 跳 4 字节
- `dest[i]` = `*(dest + i)`，步长由 dest 类型决定，与 i 的声明类型无关

### 3. 字符 vs 字符串
| | 字符 | 字符串 |
|--|------|--------|
| 写法 | 单引号 `'A'` | 双引号 `"A"` |
| 类型 | `char` | `char[]` 或 `char*` |
| 大小 | 1 字节 | N+1 字节（含 \0） |
| 用途 | 硬件命令 | 人类文本 |

### 4. 安全版 strcpy 实现（my_strcpy）
```c
int my_strcpy(char *dest, const char *src, int dest_size) {
    int count = 0;
    for(int i = 0; i < dest_size - 1; i++) {
        dest[i] = src[i];
        if(src[i] == '\0') break;
        count++;
    }
    dest[dest_size - 1] = '\0';
    return count;
}
```
- 用 `dest_size` 参数限制上限（因为数组退化为指针，函数拿不到大小）
- 永远手动 `\0` 封尾

### 5. 安全版 strcat 实现（my_strcat）
```c
int my_strcat(char *dest, const char *src, int dest_size) {
    int i = 0;
    while(i < dest_size && dest[i] != '\0') i++;  // 找到结尾
    int j = 0;
    while(src[j] != '\0' && i < dest_size - 1) {  // 留位给\0
        dest[i] = src[j];
        j++; i++;
    }
    dest[dest_size - 1] = '\0';
    return i;
}
```

### 6. 缓冲区溢出原理 🔒
- strcpy 不检查目标容量，埋头复制到见 \0 为止
- 写到 dest 后面的内存 = 覆盖相邻变量
- 攻击者可利用此机制覆盖返回地址
- 四大危险（不检查边界的）C 函数：strcpy / strcat / sprintf / gets

### 7. 其他概念
- `break` = 终止整个循环 | `continue` = 跳过本次后续
- `NULL` = 指针空值 | `'\0'` = 空字符（ASCII 0）
- 数组作为参数退化为指针，`sizeof` 在函数内返回指针大小

## 嵌入式注意事项
- ⚠️ MCU 上没有 segfault，数组越界 = 静默地篡改相邻变量/寄存器
- ⚠️ 字符串处理函数是嵌入式安全第一道防线
- ✅ 养成每次写字符串操作的代码都带上界检查的习惯

## 英语科普
- **string** = 字符串 | **character** = 字符 | **null terminator** = 空终止符
- **overflow** = 溢出 | **buffer** = 缓冲区 | **bound** = 边界
- **strcpy** = string + copy，**strcat** = string + concatenate
