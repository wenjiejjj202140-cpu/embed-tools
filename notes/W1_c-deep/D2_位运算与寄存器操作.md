# D2 位运算与寄存器操作 — 2026/7/12（下午加时）

## 位运算六种操作符

| 运算符 | 名称 | 举例 |
|--------|------|------|
| `&` | 按位与 | `0b1010 & 0b1100` = `0b1000` |
| `\|` | 按位或 | `0b1010 \| 0b1100` = `0b1110` |
| `~` | 按位取反 | `~0b1010` = `0b0101`（补码规则下注意） |
| `^` | 按位异或 | `0b1010 ^ 0b1100` = `0b0110` |
| `<<` | 左移 | `1 << 3` = `0b1000`（给第3位置1的经典写法） |
| `>>` | 右移 | `0b1000 >> 3` = `1` |

## 三板斧（寄存器操作灵魂 ⭐）

```c
// ① 置位（SET）— 把第 n 位改成 1
REG |= (1 << n);

// ② 清位（CLEAR）— 把第 n 位改成 0
REG &= ~(1 << n);

// ③ 判断（TEST）— 判断第 n 位是不是 1
if (REG & (1 << n)) { ... }
```

## 寄存器操作基础

> 寄存器 = 特殊的内存地址

```c
*(unsigned int*)0x40020C14 = 1;  // 把 1 写入地址 0x40020C14
```

**从左到右逐段拆解：**
1. `0x40020C14` — 内存地址（某个寄存器的地址）
2. `(unsigned int*)` — 把这个数字强制转换成一个指向 unsigned int 的指针
3. `*` — 解引用，操作那个地址里的值
4. `= 1` — **写操作**（不是读！）

> ⏰ 忘记 `unsigned int*` 语法的 → 回看 D3 指针笔记

## & 运算符深入

- `&` 的实质：**取地址操作**
- `int a; &a` → 得到 a 的地址值（返回值）
- `int *p = &a;` → & 在这里执行取地址的动作

## 掩码（Mask）思维

```c
// 掩码 = 用一个二进制数"过滤"感兴趣的位
#define MASK_BIT3  (1 << 3)   // 0b1000

value = REG & MASK_BIT3;      // 只保留 bit3，其他位全部归零
```

## 温故

- 0b 前缀是 C 标准里没有的（GCC 扩展），但非常直观
- `|=` 是 `REG = REG | mask` 的简写，**不是原子操作**
