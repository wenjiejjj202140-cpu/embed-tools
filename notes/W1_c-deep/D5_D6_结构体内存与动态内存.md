# D5 + D6 结构体内存 & 动态内存 — 2026/7/16

## D5 结构体内存对齐

### 结构体嵌套
- 结构体成员可以是另一个结构体
- 用 `.` 链式访问：`s.birthday.year`
- 指针用 `->` 链式访问：`p->birthday.year`

### 内存对齐规则
- 每个成员的地址必须是自身大小的倍数
  - int(4B) → 地址必须是4的倍数
  - char(1B) → 任意地址
  - short(2B) → 地址必须是2的倍数
- 结构体总大小 = 最大成员对齐值的倍数

### 成员重排省空间（面试高频）
```c
// Packed   : char + int + char = 12B
// Packed2  : int + char + char = 8B  ✅ 省4B
```

### 地址 vs 偏移
- 偏移(offset) = 相对结构体起始的距离，编译期算好
- 地址 = 硬件电路编号，走地址总线，不占存储空间

## D6 动态内存分配

### 堆 vs 栈
| 对比 | 栈 | 堆 |
|------|-----|-----|
| 分配 | 自动 | 手动(malloc) |
| 释放 | 自动(函数结束) | 手动(free) |
| 速度 | 快 | 慢 |
| 大小 | 几MB | 大部分RAM |

### malloc 标准公式
```c
int *p = (int *)malloc(n * sizeof(int));
// ① sizeof(int) → 一个int占4B
// ② n * sizeof(int) → 需要n*4字节
// ③ malloc → 向堆要空间，返回void*（通用地址）
// ④ (int *) → 强转成int*类型
// ⑤ int *p = → 存进指针变量
```

### void* 通用指针
- void = 没有类型 → void* = 没有类型信息的指针
- 可存任何类型的地址（通用容器）
- 不能解引用（不知道读多少字节）
- 需强转后才能使用

### 使用要点
- ✅ `if(p == NULL)` 检查分配是否成功
- ✅ `free(p)` + `p = NULL`（防止野指针）
- ❌ 不要 free 后继续使用（Use-After-Free）

### Valgrind 检测
```bash
gcc -o program program.c
valgrind --leak-check=full ./program
# 看 "definitely lost: 0 bytes" → 无泄漏
```

## 嵌入式注意事项
- ⚠️ 嵌入式尽量避免 malloc（碎片化/不可预测/可能失败）
- ⚠️ MCU 上内存泄漏 = 系统崩溃（没有 valgrind）
- ⚠️ 成员重排 = RAM 宝贵时的省钱技巧
- 🔒 Use-After-Free 是经典堆漏洞攻击面

## 嵌入式英语
- `align` /əˈlaɪn/ → 对齐
- `padding` → 填充字节
- `offset` → 偏移量
- `malloc` = memory + allocate
- `leak` /liːk/ → 泄漏
